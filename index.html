What is Chainer?

Chainer lets you combine multiple Claude Code plugins into automated workflows called "chains". Instead of running each plugin manually, define a chain once and execute your entire workflow with a single command.

One command replaces this entire workflow:

# Without Chainer - manual steps
/worktree-manager:start oauth
/feature-dev:feature-dev "Build OAuth authentication"
# Save spec to audit/oauth.md
/ralph-wiggum:ralph-loop
# Wait for implementation...

# With Chainer - one command
/chainer:run worktree-plan-implement \
  --feature_name="oauth" \
  --prompt="Build OAuth authentication"

Installation

git clone https://github.com/danielraffel/Chainer ~/.claude/plugins/chainer

That's it! Chainer is automatically detected by Claude Code.

Built-in Chains

plan-and-implement

Complete feature development: plan with feature-dev, implement with ralph-wiggum

/chainer:run plan-and-implement \
  --prompt="Build user dashboard" \
  --feature_name="dashboard"

worktree-plan-implement

Full workflow with isolated worktree: create worktree, plan, implement

/chainer:run worktree-plan-implement \
  --feature_name="oauth" \
  --prompt="Build OAuth2 authentication"

plan-only

Just planning, no implementation

/chainer:run plan-only \
  --prompt="Design payment system" \
  --feature_name="payments"

implement-only

Implement from an existing spec

/chainer:run implement-only \
  --spec_file="audit/oauth.md"

Create Custom Chains

Define chains in ~/.claude/chainer.local.md:

---
chains:
  my-workflow:
    enabled: true
    description: "Custom development workflow"
    inputs:
      task: { required: true, description: "What to build" }
    steps:
      - name: plan
        type: skill
        skill: feature-dev:feature-dev
        args: "{{task}}"
      - name: test
        type: script
        script: npm test
      - name: build
        type: script
        script: npm run build
---
